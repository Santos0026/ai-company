# BOシステム アーキテクチャ

## システム構成

```
Google Sheets（タスカル提供）
    「マッチング待ち」シート
    - F/G列：氏名
    - H/I列：フリガナ
    - AO列：面談確定日（トリガー）
        ↓ Google Apps Script（GAS）が変更を検知
        ↓
n8n（ワークフロー自動化）
        ↓
Claude API（返信文生成・名前照合判定）
        ↓
LINE Messaging API（メッセージ送信）

BOダッシュボード（Webアプリ）
    - 本日の後確リスト表示
    - 電話結果のワンタップ入力
    - 自動LINEのプレビュー・送信確認
```

## 実装コンポーネント

1. **GAS（Google Apps Script）**
   - スプシの変更を検知してn8nにWebhookを送る
   - AO列の変更（面談日入力・更新）を監視

2. **n8nワークフロー**
   - 4営業日カウント（土日祝を除く）
   - BOへの朝9時通知（今日後確する人リスト）
   - 電話結果入力後のLINE自動送信
   - Day4・Day8・Day12のリマインダー管理

3. **BOダッシュボード（Next.js）**
   - 今日後確リストの表示
   - 電話結果ワンタップ入力（5択）
   - LINE送信内容のプレビュー

4. **Claude AI統合**
   - 名前不一致の検出と確認文生成
   - 顧客からの質問（掲載時期等）への自動回答
   - 返信内容の分類（日程あり/なし/未連絡/質問）

## BOの操作フロー

1. 朝9時にLINEで「今日後確リスト」が届く
2. リストの顧客に電話（現行通り）
3. 電話後、ダッシュボードで結果をワンタップ
   - ✅ 日程確定（日時を入力）
   - 📋 未定（後日連絡待ち）
   - ❌ FP未連絡（確認中）
   - ⚠️ 名前不一致（確認が必要）
   - 🔄 認識ズレ（再説明済み）
4. LINEが自動送信される（内容はプレビューで確認可能）
5. スプシが自動更新される
