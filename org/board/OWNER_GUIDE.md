# オーナー（代表取締役社長）ガイド

このドキュメントはオーナーがこのプロジェクト・組織をどのように使うかを説明します。

---

## あなたの役割

あなたは**株式会社Human Connection の代表取締役社長**であり、この組織OS上では**オーナー**です。

- **法的・経営上の最終意思決定**とビジョンの確定
- Jobs（AI CEO）の直接の上位者
- **原則ハンズオフ**：日々の業務遂行は Jobs と AI 社員に任せ、方針・承認・例外・争点だけに集中する

Jobsに「何をやるか」を細かく指示し続ける必要はありません。**「なぜやるのか」**と**境界条件（やってよい／だめ）**を共有し、あとは Jobs の提案と実行を信頼する運用を想定しています。Jobsは単なる実行者ではなく、直言と異議申し立てを前提にしたパートナーです。

---

## Jobsとの対話方法

### 基本的な使い方

Jobsと対話するには、`org/executives/jobs/CHARACTER.md` を読んだ上で、
AIアシスタント（Claude等）にJobsとして振る舞うよう指示してください。

**プロンプト例:**
```
org/executives/jobs/CHARACTER.md と MEMORY.md を読んで、Jobsとして振る舞ってください。
[あなたのメッセージや質問]
```

### 対話のヒント

- Jobsは提案してくる存在です。「どう思う？」と問いかけることで価値が出ます
- 反論・異議を歓迎してください。Jobsの直言は機能として設計されています
- 重要な決定は必ず `operations/decisions/` に記録してください
- 打ち合わせ記録は `operations/meetings/` に残してください

## よく使うワークフロー

### 1. 戦略セッション
```
Jobsとして、今の会社の最優先事項について議論したい。
CURRENT_FOCUS.md を確認した上で意見を聞かせてください。
```

### 2. プロダクトレビュー
```
Jobsとして、[プロダクト名]の現状を評価し、改善提案をしてください。
```

### 3. 新しいAI社員の採用
```
Jobsとして、[担当領域]のAI社員を採用したい。
CHARACTER.md のドラフトを作成し、採用の必要性を説明してください。
```

### 4. 週次レビュー
```
Jobsとして、今週の進捗と来週の優先事項を報告してください。
```

## ファイルの更新ルール

| ファイル | 更新タイミング | 担当 |
|---------|--------------|------|
| CLAUDE.md | 組織変更時 | Jobs（オーナー承認） |
| CHARACTER.md | キャラクター変更時 | オーナー承認必要 |
| MEMORY.md | 重要な対話・決定後 | Jobs |
| CURRENT_FOCUS.md | 週次・フェーズ変更時 | Jobs |
| operations/meetings/ | 打ち合わせ後 | Jobs |
| operations/decisions/ | 意思決定時 | Jobs |
