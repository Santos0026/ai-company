# 情報連携の断絶問題と改善案：2026年4月

**作成**: 2026-04-17 by Jobs

---

## 発見した問題

インタビュアー → BO への情報が引き継ぎされていない。

現状の情報フロー：
```
インタビュアー
  ↓ 決める
  ・FP相談テーマ（例：老後資金、子どもの教育費）
  ↓ 共有されない ❌
バックオフィス（BO）
  ↓ 知らないまま後追い電話
  ・「当日はリラックスして」とも言えない
  ・相談テーマを言及できない
  ↓
FP面談当日
  ・掲載者が「何を話せばいいか」曖昧なまま臨む
  ・面談の質が下がる → 継続率・承認率に影響
```

---

## 影響範囲

現在の継続面談率60%・承認率80%は、情報連携なしで達成している数字。
相談テーマが引き継がれていれば、両指標がさらに上がる可能性がある。

また、BOが掲載者の背景を知らないまま後追い電話をしているため：
- 「〇〇について相談するんでしたよね」という共感トークができない
- 掲載者ごとの温度感・懸念点を把握できない
- 対応がどうしても画一的になる

---

## 解決策：最小コストで今すぐできる方法

### 案A：Googleスプレッドシートで共有管理表を作る（推奨）

インタビュアーがインタビュー終了後5分以内に入力。
BOが後追い前に確認する。

**管理表の項目（最低限）：**

| 項目 | 例 |
|------|-----|
| ヒトモノ番号 | A0300 |
| 掲載者名 | 山田太郎 |
| 業種・活動内容 | カフェ経営 |
| 選んだアンケート | FP（タスカル・04番） |
| 相談テーマ | 老後資金・現在の保険の見直し |
| FP日程ステータス | 未確定 / 〇月〇日〇時確定 |
| 特記事項 | FPへの抵抗感なし・積極的 |
| 担当インタビュアー | 藤堂 |

**運用ルール：**
- インタビュアー：終了後5分以内に入力
- BO：後追い電話の前に必ず確認
- BOが後追い後にFP日程ステータスを更新

---

### 案B：インタビューLINEのトーク欄に入力（即時性重視）

インタビュー終了後、インタビュアーが掲載者とのLINEトーク欄に
スタッフ向けメモをスタンプで仕切って入力する。

例：
```
[スタッフメモ]
アンケート：FP04（タスカル）
相談テーマ：子どもの教育費・老後
温度感：高（積極的）
特記：保険業界ではない・FP面談に前向き
担当：藤堂
```

BOはこのメモを見てから後追い電話をかける。

**メリット：** ツール追加なし、今日から始められる
**デメリット：** LINEトーク内が混在する・検索性が低い

---

## Jobs推奨：案Aのスプレッドシート

理由：
1. 件数が月100件規模になると、LINEメモでは管理が追いつかない
2. 日程ステータスの一覧管理ができる（「未確定リスト」を毎朝BOが確認できる）
3. 将来的に分析・改善のデータ源になる
4. Googleスプレッドシートなら無料・全員スマホで使える

---

## 実装ステップ（最短1日で完成）

1. Googleスプレッドシートを作成（Santosまたは Jobs が設計）
2. インタビュアー2名・BO1名にシェア
3. 入力ルール・タイミングをチームに周知（口頭5分でOK）
4. 翌日から運用開始

---

## 期待効果

- FP面談当日の「何を話せばいいか分からない」が解消
- BOが個別の温度感を把握した上で後追いできる
- FP日程の進捗をリアルタイムで全員が把握できる
- 継続面談率60% → 70%以上を目標
- 月間売上換算：+数万〜十数万円（継続率向上分）
