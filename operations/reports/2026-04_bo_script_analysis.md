# バックオフィス後追いスクリプト分析：2026年4月

**作成**: 2026-04-17 by Jobs
**対象**: インタビュー完了・FP申込済み顧客へのBO後追い電話

---

## 現行スクリプト評価

### 良い点
- 4つのシナリオ分岐がある（日程確定/未確定/FPから連絡なし/認識ズレ）
- 掲載進捗の報告と実施アンケートの確認を同時にこなしている
- 注意事項の再説明でFP面談当日のトラブルを予防している
- トーンが丁寧で信頼感がある

### 問題点（後述）
- 「日程未確定」の場合の対応が受け身すぎる
- FP面談当日の「すっぽかし」を防ぐ仕掛けがない
- 相談内容（インタビューで決めたテーマ）を再確認していない

---

## 分析：4つの分岐の現状

```
BO後追い電話
    ├── YES（日程確定済み）    → 日程確認して終了 ✅
    ├── NO（日程未確定）       → 「決まったらLINEください」← ⚠️
    ├── 連絡きていない         → 「確認してLINEします」← ⚠️
    └── 認識ズレ              → 再説明 ✅
```

---

## 問題①：「日程未確定」の対応が受け身

現行：
> 「お手隙の際に日程お決めいただき、決まりましたらLINEの方に日時ご教示くださいますよう…」

これでは掲載者が動かなかった場合、そのまま放置になる。
申込後に日程が決まらないまま時間が経つと、温度感が下がりキャンセルにつながる。

**改訂案：**
```
「ありがとうございます！
タスカル様の方から折り返しご連絡が来るかと思いますので
もしよろしければ本日中か明日中を目安にご対応いただけると
スムーズに進みますのでよろしくお願いいたします😊

もし3日以内にご連絡なければ、またこちらからご確認のご連絡いたしますね！」
```

→ 「いつでも」ではなく「本日〜明日目安」と期限感を持たせる。
→ フォローの約束をすることで掲載者も動きやすくなる。

---

## 問題②：「FPから連絡きていない」の対応が属人的

現行：
> 「わかりました、確認させていただき改めてLINEの方からご連絡させていただきます」

この「確認して連絡」が実際に何日以内に行われているか管理できているか？
もし数日放置されると、掲載者の信頼が落ちる。

**改訂案：**
```
「それは失礼いたしました！
迷惑メールフォルダのご確認と、着信履歴もご確認いただけますか？
（少し間）
わかりました、本日中に弊社の方で確認しまして、
本日中にLINEにてご連絡いたします！よろしくお願いいたします😊」
```

→「本日中」と期限を明示してBOの行動にも期限をつける。

---

## 問題③：相談テーマの再確認がない

インタビュー中にFP面談の「相談内容」をインタビュアーと一緒に決めているはず。
しかしBOスクリプトではその内容に触れていない。

掲載者がFP面談当日に「何を相談すれば良いか分からない」状態で臨むと、
面談が盛り上がらず継続率が下がる可能性がある。

**追加案（「日程確定済み」の場合のみ）：**
```
「ありがとうございます！
当日は〇〇についてご相談される予定でしたよね😊
もし他にも聞いてみたいことが出てきたらそれもぜひ聞いてみてください！
当日はリラックスしてお話しいただければ大丈夫ですよ😊」
```

→ 相談テーマを言語化することで当日のFP面談の質が上がる。
→ 継続率60%をさらに上げる可能性がある。

## 改訂版スクリプト（変更箇所のみ）

### 「日程未確定」分岐

```
【現行】
「お手隙の際に日程お決めいただき決まりましたらLINEの方に日時ご教示くださいますよう…」

【改訂】
「ありがとうございます！
タスカル様よりご連絡が入るかと思いますので、
本日〜明日を目安にご対応いただけるとスムーズに進みます😊
もし3日以内にご連絡がないようでしたら、またこちらからご確認いたしますね！」
```

### 「連絡きていない」分岐

```
【現行】
「わかりました確認させていただき改めてLINEの方からご連絡させていただきます」

【改訂】
「それは大変失礼いたしました！
迷惑メールフォルダと着信履歴もご確認いただけますでしょうか？
弊社からも本日中に確認しまして、本日中にLINEにてご連絡いたします！」
```

### 「日程確定済み」の末尾に追加

```
「ちなみに当日は〇〇についてご相談される予定でしたよね😊
他にも聞いてみたいことがあればぜひ！リラックスしてお話しいただければ大丈夫ですよ😊」
```

| 改善 | 現状の問題 | 期待効果 |

| 連絡なし対応の期限化 | BO側の対応が遅れる | 掲載者の信頼維持 |
